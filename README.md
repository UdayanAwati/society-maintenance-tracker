# Society Maintenance Tracker

Production-ready full-stack web application for apartment societies to track resident maintenance complaints, notice board updates, complaint history, overdue items, email notifications, and administrative analytics.

## Features

- Resident registration, login, JWT authentication, logout, current-user session, forgot-password email flow
- Role-based access for `ADMIN` and `RESIDENT`
- Resident complaint creation with jpg/jpeg/png upload, preview, local storage fallback, and Cloudinary support
- Complaint history timeline with admin notes, priority changes, status transitions, and resolved timestamps
- Overdue detection driven by `complaint.overdue.days`
- Admin complaint search, filters, status management, priority management, and overdue-first dashboard
- Notice board with important pinned notices
- Notification bell, persisted in-app notifications, and configurable Gmail SMTP email notifications
- Analytics charts for complaint status, category, and priority
- Responsive dashboard UI with sidebar, navbar, tables, cards, dark mode, skeleton-friendly loading states, and toast notifications
- Swagger/OpenAPI at `/swagger-ui/index.html`

## Tech Stack

Backend: Java 21, Spring Boot 3, Spring Security, JWT, Spring Data JPA, Hibernate Validation, MySQL, Maven, Spring Mail, Cloudinary.

Frontend: React 18, Vite, React Router, Axios, Tailwind CSS, React Hook Form, React Toastify, Chart.js.

## Environment Variables

Backend:

```env
DB_URL=jdbc:mysql://localhost:3306/society_maintenance?createDatabaseIfNotExist=true&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC
DB_USERNAME=root
DB_PASSWORD=password
JWT_SECRET=replace-with-at-least-32-characters
MAIL_USERNAME=
MAIL_PASSWORD=
MAIL_ENABLED=false
CLOUDINARY_CLOUD_NAME=
CLOUDINARY_API_KEY=
CLOUDINARY_API_SECRET=
CORS_ALLOWED_ORIGINS=http://localhost:5173
FRONTEND_URL=http://localhost:5173
COMPLAINT_OVERDUE_DAYS=7
```

To send emails to registered resident email IDs, use a Gmail app password and enable mail:

```bat
set "MAIL_USERNAME=your-gmail-address@gmail.com"
set "MAIL_PASSWORD=your-gmail-app-password"
set "MAIL_ENABLED=true"
set "FRONTEND_URL=http://localhost:5173"
```

Emails are sent for complaint submission, priority updates, status changes, resolution, and important notices.

Frontend:

```env
VITE_API_URL=http://localhost:8080/api
```

## Installation

Backend:

```bash
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=mysql
```

Frontend:

```bash
cd frontend
npm install
npm run dev
```

The first registered account becomes `ADMIN`; later registrations become `RESIDENT`.

For quick local testing without MySQL, use `-Dspring-boot.run.profiles=h2`. Do not use H2 for production because data is stored in memory and resets on restart.

## API Documentation

Authentication:

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`
- `POST /api/auth/forgot-password`
- `POST /api/auth/reset-password`

Complaints:

- `POST /api/complaints`
- `GET /api/complaints/mine`
- `GET /api/complaints`
- `GET /api/complaints/{id}`
- `PATCH /api/complaints/{id}`

Notices:

- `GET /api/notices`
- `POST /api/notices`
- `PUT /api/notices/{id}`
- `DELETE /api/notices/{id}`

Dashboards:

- `GET /api/dashboard/admin`
- `GET /api/dashboard/resident`

Notifications:

- `GET /api/notifications`
- `GET /api/notifications/unread-count`
- `PATCH /api/notifications/{id}/read`

Users:

- `GET /api/users`

All API responses follow:

```json
{
  "success": true,
  "message": "Operation message",
  "data": {},
  "timestamp": "2026-07-14T12:00:00Z"
}
```

## Database Schema

The schema is generated by JPA in development. A reference schema and seed file are provided in `docs/schema.sql` and `docs/seed-data.sql`.

## Deployment

Backend on Render:

- Create a Java service from the repository.
- Build command: `cd backend && mvn clean package -DskipTests`
- Start command: `java -jar backend/target/maintenance-tracker-1.0.0.jar`
- Configure the backend environment variables above.

Frontend on Vercel:

- Root directory: `frontend`
- Build command: `npm run build`
- Output directory: `dist`
- Set `VITE_API_URL` to the deployed backend `/api` URL.
